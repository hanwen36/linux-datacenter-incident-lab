# Linux Data Center Incident Lab

This repository contains hands-on Linux troubleshooting labs designed to simulate real-world incidents that commonly occur in data center environments.

The goal of this project is to practice diagnosing and resolving system issues using standard Linux tools and operational runbooks.

## Environment

Ubuntu Server (VirtualBox)

## Skills Demonstrated

Linux system troubleshooting  
Disk usage analysis  
Log investigation  
Incident response workflow  
Root cause analysis  
System recovery verification

## Tools Used

df  
du  
lsof  
iostat  
sort  
rm  

## Lab Projects

P1 – Disk Full Incident  
P2 – Inode Exhaustion  
P3 – Deleted File Handle  
P4 – High IO Incident  
P5 – Log Explosion  
P6 – RAID Rebuild

Each project includes:

- Incident description
- Investigation steps
- Runbook procedures
- Terminal command logs
- Screenshots of troubleshooting process

## Purpose

These labs are built to simulate common operational incidents faced by Data Center Engineers and Site Reliability Engineers, and to document the troubleshooting process in a structured and reproducible way.
